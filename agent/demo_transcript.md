# 对话式反诈 Agent · 演示对话

> 这是 `agent/guardian_agent.py` 配置 API key 后的真实运行形态示例（以"南开 220 万"同类情形为例）。
> 它体现的是"Agent"而非"分类器"：**会追问、会调用案例库工具查证、最后给可执行处置**。
> 配好 key 后 `python agent/guardian_agent.py` 即可实时对话复现。

---

**你>** 我在闲鱼买个相机，对方客服说我操作错了被多扣了钱，要帮我退

**安全官>** 别急，我帮你看看。先确认两件事：
1. 对方是让你**点链接、扫码**，还是要你**下载一个软件、开"屏幕共享"**来帮你操作退款？
2. 你现在**操作到哪一步了**——有没有已经转账、或者把验证码/银行卡信息给对方？

**你>** 他让我下了个会议软件开了共享屏幕，说要核对流水，我还没转钱但他在指导我操作

**安全官>** 　〔查案例库：闲鱼 退款 共享屏幕 刷流水… → 命中 KB-029, KB-004〕

**立即停下，现在就关掉共享屏幕、退出那个会议软件。** 这是高风险——典型的「共享屏幕／远程控制类诈骗」。

为什么判断是诈骗：
- 国家反诈八个凡是：凡是自称客服、以退款理赔为由要你**开共享屏幕**的，都是诈骗。
- "共享屏幕"会让对方实时看到你的银行卡余额、短信验证码——他"指导你操作"的下一步，就是把钱转走或诱导你转账。
- 正规电商退款**原路退回**，绝不需要你下载软件、开屏幕共享、报流水。

现在请按顺序做：
1. **立即关闭共享屏幕、卸载刚下载的软件**，断开和对方的联系。
2. 检查手机里有没有被诱导开通的免密支付/转账，**关掉**。
3. 拨打 **96110** 反诈专线说明情况；校内可找**南大保卫处 81686110**。
4. 如果已经有任何转账，立刻打 **110** 报警并向银行挂失止付，保留聊天和转账记录。

你刚才说还没转钱——很好，及时停手就没事。要不要我帮你把这条提醒整理成一段话，发到班级群提醒同学？最近就有高校学生因为同样的套路被骗了大额资金。
