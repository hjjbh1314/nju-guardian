01 — 痛点

通用反诈，接不住校园场景

  • 壹类型繁多，难以全记高发诈骗类型数十种且持续翻新，仅凭个人记忆难以全面识别与防范
  • 贰校园场景命中率低冒充辅导员、奖助学金、保录留学、二手交易等校园高发场景，通用工具难以精准识别
  • 叁传统宣教触达有限图文手册阅读率偏低，学生更需要「即时输入、即时研判」的轻量工具
  • 肆案例覆盖与时效案例的覆盖广度与更新时效，是同类知识库普遍面临的瓶颈

05 — 技术内核

分层研判：三路引擎 ＋ LLM Agent

用户输入→ 三路并行召回→ 加权融合 + 校准→ 研判卡片→ 难例升级 LLM Agent

① 端侧三路引擎 · 毫秒 / 离线 / 可解释

  • 规则路　关键词 + 正则，强命中即定高危
  • 语义路　BGE-zh 向量，覆盖「换个说法」变体
  • 行为红旗　抽反诈红线要素（转账/验证码/共享屏幕/冒充…）——库中无此类型也能预警

② LLM Agent · 难例升级（DeepSeek）

  • 多轮追问关键细节
  • 调用案例库工具查证
  • 给出按优先级的处置方案

风险定级 ＝ 案例研判 与 行为研判 取更高者　——　已知套路、未知变体都不漏

案例研判：综合分 ＝ 规则 ＋ 2.5× 语义　·　行为研判：红旗 强3 / 中2 / 弱1 计分，≥2 强或累计 ≥6 判高危　·　缺模型自动降级 TF-IDF

07 — 核心进步

一条会自己长大的流水线

公开来源→ 抓取→ LLM 结构化→ 格式校验→ 去重→ 人工审核 PR→ 入库

前五步全自动——现场可一键触发，也支持 GitHub Actions 定时采集（按需开启）。

关键的一道关是人工审核：

  • 由人复核事实、核验公开来源后，案例方可入库
  • 保证每条案例可追溯、不污染知识库
  • 这正是它区别于无脑爬取、能长期可信的原因
