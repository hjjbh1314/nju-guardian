# 南大数智安全官 · 答辩问答清单

> 评委高频问题 + 可直接念的答法。原则：**先正面答，再主动说局限**（主动暴露局限反而加分）。

## 一、技术原理

**Q1：为什么是"三路"，三路怎么融合？**
规则路（关键词+正则，毫秒级可解释）、语义路（BGE 中文向量，覆盖"换个说法"）、行为红旗（抽取转账/验证码/共享屏幕等通用诈骗行为）。融合方式：案例排序用"规则强度 + 2.5×语义相似度"；**风险定级取"案例研判"和"行为研判"中的更高者**——已知套路靠前两路，未知变体靠行为红旗，都不漏。

**Q2：为什么不直接用大模型 / 更大的模型？**
这是刻意的**分层设计**：端侧三路引擎做毫秒级、可离线、可解释的秒判，只有难例才升级到大模型。好处是成本可控、响应快、隐私可端侧。无脑全上大模型，对一个要装进学生手机、随时要用的工具反而是错的。嵌入模型 BGE-small 对我们这个量级检索已经够用，Top-3 召回已 100%，没有提升空间可榨。

**Q3：行为红旗会不会误报？**
正常文本零信号（实测）；而且它是**抬升**风险、不是唯一判据，与案例研判取高。强信号有阈值校准（≥2 个强信号或累计严重度≥6 才判高危）。

**Q4：检测的可解释性？**
每个判断都给"命中了什么词/什么行为 + 为什么可疑 + 对应反诈预警要点 + 公开来源"，不是黑箱打分。反诈场景必须可信、可追溯。

**Q5：模型或网络挂了怎么办？**
自动降级 TF-IDF，永不空转；端侧规则路本身就离线可用。

## 二、数据与市场测算

**Q6：市场测算的发案率 0.5%、拦截率 20% 怎么来的，不是拍的吧？**
都有基准、且从严取值。**年发案率 0.5%**：调查显示近 6 成大学生收到过诈骗信息，我们按其中约 1% 真正造成损失保守折算。**拦截率 20%**：用我们实测的识别能力（Top-3 和泛化召回都接近 100%）乘一个保守的 20% 用户渗透率——识别强，但我们不假设人人都来查。

**Q7：人均损失 7500 元可靠吗？**
是浙江某高校 2018–2023 年公开通报的 215 起、合计 162 万元算出来的真实校园样本，比全国平均更贴校园场景。

**Q8：案例库的数据可信吗？会不会爬来的有错？**
只取政府、央媒、高校保卫处的**公开来源**，必须改写不照抄、来源可核验；入库前**强制人工审核**。这正是它区别于无脑爬取、能长期可信的原因。

**Q9：50 类够吗？**
够覆盖校园高发场景，且系统**会自己长大**——公开来源自动采集 + 现场一键上报，用得越多库越全。数量不是重点，"有治理、可增长的机制"才是。

## 三、产品与隐私

**Q10：怎么保证交给大模型的数据是安全的？**（必问）
①默认端侧检测、不上传；只有用户**主动点 AI 研判**才传**那一条**文本。②数据最小化，不含身份、通讯录、设备信息。③经我们的加密代理转发，DeepSeek 密钥存在**服务端 Secret**，前端看不到。④代理**无状态、不留存、不入库、不记日志**（已核实）。⑤生产环境可平替为**校内私有化部署/本地模型**，数据不出校。
> 别说满：数据确实经过 DeepSeek 这个第三方；用"受其条款约束 + 可私有化部署"兜，别说"绝不经过任何第三方"。

**Q11：现场用微信扫码打不开/很慢？**
H5 挂在 GitHub Pages、AI 研判走 Cloudflare，国内访问需要正常网络环境。但**核心检测是纯前端、可离线**，我们也准备了"不扫码也能看"的演示页和本地版，现场不依赖网络也能完整演示。

**Q12：和国家反诈中心 App 有什么区别？不会重复造轮子吗？**
我们是**校园场景特化 + 即时研判**的补充，不是替代：聚焦冒充辅导员、奖助学金、二手交易等通用 App 命中率低的场景；交互是"粘进去秒出卡片"，更轻、更快。

## 四、落地、商业与团队

**Q13：怎么复制到别的学校？成本多少？**
开源 + 自增长，每复制一所高校几乎**零边际成本**——只要把它的校园场景案例喂进同一条流水线。年运营成本不到 10 万。

**Q14：商业模式 / 可持续性？**
定位是**公益+社会价值**优先：开源、零边际复制；可持续靠"自增长流水线 + 高校/保卫处共建"。不靠收费，靠把损失留在学生口袋里。

**Q15：自动采集是真在跑吗？**
采集、结构化、校验、去重的脚本都已实现并实测；定时自动跑用 GitHub Actions，目前是**可手动触发演示 + 可按需开启定时**，能力具备、不强求天天跑（也是成本考虑）。

**Q16：现场能演示自动化吗？**
能。Actions 里点一下 Run workflow，就能看它从采集跑到自动开 PR，日志可见。

## 五、风险与诚实（主动说，加分）

**Q17：最大的局限是什么？**
两点：①AI 研判依赖第三方大模型，隐私上靠"不留存+可私有化"兜底，更高要求需自建模型；②市场测算的发案率/渗透率是保守假设，量级稳健但非精确值。我们对这两点都做了从严处理。

**Q18：会不会有人拿它来钻空子/对抗？**
行为红旗对"换个说法"的变体有泛化能力；对抗是长期攻防，靠案例库持续更新 + 人工审核迭代。

**Q19：这套东西的核心壁垒到底是什么？**
不是某个模型，而是**"会自己长大的、有治理的反诈知识系统"**——三路融合 + 自增长流水线 + 人工审核护栏，数据和规则越用越强。
