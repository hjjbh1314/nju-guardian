# 南大数智安全官 · 决赛冲刺两周计划

> 目标：把案例库从「一次性填的 50 条静态资产」升级为「带格式治理、可自动采集、可现场展示的活系统」。
> 这是从初赛到决赛最有说服力的进步叙事：**不是堆数量，是建了一套有治理、可追溯、会自增长的知识生产机制。**

技术路线（已定）：
- 现场展示 = **H5 移动端先行**，微信小程序 / 南大 app 对接作为路线图
- 自动更新 = **GitHub Actions 定时**（云端 cron，自动开 PR，运行日志可截图给评委）

核心流水线（贯穿全程）：

```
公开来源(政府/央媒/高校保卫处公开推文)
  → 抓取原文 → LLM 结构化成 12 字段 → validate_kb.py 格式校验 → 去重比对
  → 落到 candidates/ 暂存区 → 自动开 PR → 人工一键审核合并
```

## ⑤ 还能进步的方向（超出两周、但答辩可"画饼"）

| 方向 | 一句话 | 为什么打动评委 |
|---|---|---|
| 反馈闭环 | 用户上报 → 进暂存区 | 冷启动的根本解法：越用越强 |
| 评测指标 | 命中率/召回率量化 | 从"我觉得好"到"数据证明好" |
| 可解释性 | 每个判断都给"为什么+来源" | 反诈场景必须可信、可追溯 |
| 隐私优先 | 检测尽量端侧/不留存输入 | 处理诈骗话术=敏感数据，是加分项 |
| 多轮对话 | 追问"对方还说了什么" | 比单次匹配更像真助手 |
| 南大场景特化 | 教务/迎新/兼职等校内高发 | "接入南大 app"的现实落点 |
| 家长/新生版 | 分人群话术 | 受众扩展，社会价值叙事 |
