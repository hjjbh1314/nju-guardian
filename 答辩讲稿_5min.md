## 第 1 页 · 封面（10 秒）

> 评委老师下午好/晚上好，我是南京大学经济学拔尖计划本科生作者。本次汇报的项目是「南大数智安全官 NJU Guardian」——基于 OpenClaw 平台的校园场景化反诈 AI Agent。下面用五分钟向各位老师汇报。

---

## 第 2 页 · 设计目的（35 秒）

> 本项目的立项出发点是一个明确的覆盖空白。
>
> 国家反诈中心 App 在通用场景下已经较为成熟；但是辅导员收班费、奖助学金 ATM 激活、新生兼职刷单、老乡群高仿账号，以及最近高发的虚假校园招聘——这些发生在校园语境里的话术，在通用产品上命中率明显偏低。
>
> 我们想回答的问题是：能不能做一款学生「粘进去就出结果」的工具，作为国家反诈 App 在校园场景的补充应用，并在长期嵌入南大官方 App。

---

## 第 3 页 · 核心思路 + 系统架构（55 秒）

> OpenClaw 做平台，我们做校园内容——所有 OpenClaw 已经做得了的事，本项目都不重做一遍。整套系统是三层架构。
>
> 用户层在微信小程序和 H5 上，支持三种输入入口：粘贴可疑文本、上传截图走 OCR、直接输入可疑链接或电话号码。
>
> 中间层部署在 OpenClaw 平台。OpenClaw 三项核心能力在系统里各有分工——
> 多模态理解模块承担截图 OCR、二维码识别与转账界面识别；
> 工具调用模块承担 URL 黑名单查询、向量库检索、外部反诈接口与脱敏；
> 流程编排模块承担主控 Agent 与子 Agent 的协同调度，以及案例回流的自动化。
>
> 底层是诈骗知识库，由向量检索引擎和结构化案例库共同支撑，所有数据来源于国家反诈手册、公安部公开预警、央视与新华社通报，以及南京大学保卫处反诈材料。

---

## 第 4 页 · OpenClaw 多 Agent 编排（60 秒）★ 重点

> 这一页讲 OpenClaw 上的 Agent 怎么分工。系统编排了五个 Agent 和 Workflow——主控派活，三路子 Agent 并行干。
>
> 主控 Agent 识别输入类型并完成路由，调用的是 OpenClaw 的流程编排能力。
>
> 三个子 Agent 并行处理——文本风险 Agent 执行 RAG 检索与规则匹配，完成校园话术分类；多模态截图 Agent 执行 OCR、二维码与场景识别；链接电话 Agent 执行 URL 风险接口调用、短链还原与号码归属查询。
>
> 识别完成后，聚合输出工作流把三路信号融合成结构化卡片；案例回流 Workflow 在用户授权之后由脱敏 Agent 自动处理姓名、手机号、银行卡号等敏感字段，然后写入知识库。这一步对项目长期价值非常关键——本校真实案例能持续回流，反哺学工部与保卫处的反诈预警。
>
> 整张架构图的设计原则是：所有能由 OpenClaw 原生承担的环节，本项目都不重复开发，团队精力集中在内容与叙事。

## 第 5 页 · 双路融合原型（35 秒）

> 这一页交代当前进展。
>
> 系统检测引擎是规则路加向量路的双路融合。规则路用 keywords 和 regex 命中得到 rule_score；向量路用 BGE-zh 中文优化嵌入算余弦相似度得到 vector_sim。综合分是 rule_score 加上 2.5 倍的 vector_sim。
>
> 当前知识库覆盖 50 类——国家反诈中心十大类、AI 换脸与共享屏幕等新型五类、冒充辅导员与虚假校招等校园特化十二类。每条案例都强制带 source 字段引用公开来源；8 条标杆输入的冒烟测试 TOP-1 全部命中正确 case。
>
> 这套引擎是 OpenClaw 多 Agent 编排在本地的工程预演，决赛版本以原生 RAG 形式承担同一职能。

## 第 6 页 · 现场 Demo · token 流转（65 秒）★ ★ 重点

> 下面以一条典型场景演示 OpenClaw 平台的端到端调用过程。
>
> 输入：「辅导员让我代收班费，要我先转 800 给他他周一返我。」
>
> 第一步，OpenClaw 收到输入后由主控 Agent 解析为文本类型，消耗输入 token。
>
> 第二步，主控 Agent 通过流程编排能力把任务路由到文本风险 Agent，平台日志同步落盘，消耗编排 token。
>
> 第三步，文本风险 Agent 走 RAG 通道——BGE-zh 嵌入加向量检索，从知识库中召回 KB-008「冒充辅导员」，消耗嵌入与检索 token。
>
> 第四步，工具调用接口对文本中的 URL 和电话号码做黑名单查询；本例没有 URL，跳过该工具，但这一调度路径在平台上完整记录。
>
> 第五步，聚合输出工作流生成🔴高风险卡片：综合分 4.2，命中反诈预警要点第 6 条，三步建议——立即停止转账并核实身份、用电话二次反向核实、已转账拨打 96110 或南大保卫处 81686110。
>
> 整个过程在 OpenClaw 平台上的 token 流转和计费完全透明，便于团队迭代和成本控制。

> 隐私边界从立项就守住四条线：即用即弃、用户授权回流、案例可追溯、只识别不替代决策。

## 第 9 页 · 价值测算 · 成本与挽损（25 秒）

> 最后从数据角度回答"为什么值得做"。
>
> 2024 年国家反诈中心拦截涉案资金 3,151 亿元，全国破获电诈案件 29.4 万起；南京大学在校学生 47,911 人，受害大学生中约 20% 损失超过 5,000 元——这是南都研究院 2021 年的调研口径。
>
> 本系统覆盖公安部 TOP3 类型，按保守的 30% 早期干预率测算，每年可挽回的损失在 50 到 100 万元之间。成本方面，单次检测 token 边际成本约 0.015 元；含云服务和域名在内的年综合成本控制在 5,000 元以内。
>
> 投入产出量级在 100 倍以上，且不计声誉、合规与教育外溢。
